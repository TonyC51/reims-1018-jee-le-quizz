Backlog

Le backlog consiste à recueillir tous les besoins du client que l’équipe projet doit réaliser. Il contient donc la liste des fonctionnalités intervenant dans la constitution d’un produit, ainsi que tous les éléments nécessitant l’intervention de l’équipe projet. Tous les éléments inclus dans le backlog scrum sont classés par priorité indiquant l’ordre de leur réalisation.

Via Trello, nous avons mis en place notre backlog pour ce projet, les users stories et les sprints y sont inscrits.

Semaine2

Sprint 1

Lors de ce deuxième sprint nous avions pour but de réaliser quatre user stories :

US# 2 : Identification visiteur

En tant qu' utilisateur, je veux entrer mon pseudo, afin de pouvoir m'identifier.

Lorsque l'utilisateur éxécute le fichier quizz et arrive sur le menu principal du jeu, un message s'affiche précisant d'entrer le pseudo désiré et de taper sur la touche entrée pour le valider

Tests de validations : Je peux rentrer mon pseudo sur la page d’accueil.

US# 3 : Lecture du concept et des règles du jeu

En tant qu'utilisateur, je veux lire les instructions et les règles du jeu, afin de comprendre les règles du quizz. Elles sont présente et mise en page de façon lisible dans le terminal

Test de validation :-Je peux accéder à une option me permettant de consulter toutes les règles du jeu.

US# 4

En tant qu'utilisateur, je veux pouvoir commencer la partie à l'aide d'une commande à taper, qui m'est communiquée par la console sous forme de message, afin de débuter une partie rapidement.

Test de validation : je reçois le message m'invitant à taper play dans la console puis a valider avec entrée, lorsque je tape la mauvaise commande je reçois un message m'invitant à réessayer et le curseur d'entrée clavier est toujours présent. Si je tape la bonne commande, je reçois un message "here we go" et la partie commence.

US# 5 : Réalisation des questions

Réalisation d'un fichiez JSON contenant les différentes questions du quizz ainsi que les réponses attendues.
